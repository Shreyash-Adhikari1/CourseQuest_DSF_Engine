# Cybersecurity Pathway
## BSc (Hons) Ethical Hacking and Cybersecurity

---

## Course Overview

This pathway provides technical, analytical, and ethical training in protecting digital systems and data. It covers the full spectrum of cybersecurity — from foundational principles and legal frameworks through to advanced offensive security, penetration testing, digital forensics, and applied cryptography.

The program is structured so that students first understand how systems work before learning how to exploit and protect them. Year 1 establishes computing and security fundamentals. Year 2 moves into practical offensive and defensive skills. Year 3 focuses on advanced attack simulation, reverse engineering, exploit development, and independent research.

This is a technically demanding, hands-on pathway. Students spend significant time in lab environments using real security tools — Kali Linux, Wireshark, Metasploit, Ghidra, and others — working on realistic attack and defence scenarios.

---

## What You Will Study — Year by Year

### Year 1 — Foundations
- **Ethics and Legal Framework of Cybersecurity** — GDPR, computer misuse legislation, cybercrime law, ethical hacking boundaries, professional conduct, ISO/IEC 27001
- **Introduction to Programming** — Python fundamentals, variables, loops, functions, file handling, basic data structures, logical problem solving
- **Foundations of Computer Science** — Logic, algorithms, number systems, binary arithmetic, Boolean algebra, computational complexity
- **Networking and Computer Architectures** — OSI and TCP/IP models, IP addressing, routing, CPU architecture, Cisco Packet Tracer, GNS3
- **Information Security Management** — Security policy development, risk management, incident response, ISO 27001 frameworks
- **Cyber Security Fundamentals** — Threat types, malware, encryption basics, authentication, CIA triad (Confidentiality, Integrity, Availability), Wireshark, Kali Linux basics

### Year 2 — Applied Security Skills
- **Internet and Web Technologies** — Web development, relational databases, web application security architecture, secure programming
- **Digital Forensics** — Evidence recovery, deleted file analysis, forensic investigation procedures, FTK Imager, Autopsy, Volatility, Wireshark
- **Programming and Operating Systems** — Linux OS, memory management, process scheduling, virtual memory, system calls, C programming
- **Security Operations** — SOC management, real-time threat monitoring, incident response, Splunk, ELK Stack, Security Onion
- **Foundations of Networking** — Advanced networking protocols (RIP, OSPF, BGP), LAN/WAN/MAN, hands-on with real Cisco hardware
- **Practical Penetration Testing** — Penetration testing methodology, exploitation techniques, security auditing, underlying systems theory

### Year 3 — Advanced Offensive and Defensive Security
- **Digital Security Risk and Audit Management** — Risk identification, mitigation strategies, security audits, compliance, Nessus, OpenVAS, NIST framework
- **Reverse Engineering and Exploit Development** — Binary analysis, exploit writing, shellcode, GDB, Ghidra, IDA Pro, malware analysis, de-obfuscation
- **Research Project Preparation** — Research methodology, topic selection, proposal writing, literature review
- **Advanced Penetration Testing** — Red teaming, advanced attack simulation, post-exploitation, Cobalt Strike, BloodHound, Metasploit Pro
- **Applied Cryptography** — AES, RSA, digital signatures, hashing algorithms, secure communication protocols, OpenSSL
- **Research Project Delivery** — Final year independent research project with technical implementation and written report

## Career Opportunities

Graduates of this pathway are prepared for roles including:

- **Cybersecurity Analyst** — monitoring systems and responding to threats
- **Penetration Tester / VAPT Specialist** — simulating attacks to identify vulnerabilities
- **Ethical Hacker** — authorised offensive security testing
- **Offensive Security Engineer (Red Team)** — advanced attack simulation
- **Digital Forensics Analyst** — investigating digital evidence and cybercrime
- **Security Incident Responder / Threat Hunter** — detecting and responding to active threats
- **Security Operations Center (SOC) Analyst** — real-time monitoring and response
- **Malware Analyst** — reverse engineering malicious software
- **Network Security Engineer** — securing network infrastructure
- **Information Security Manager** — managing organizational security posture
- **System Auditor** — assessing compliance and security controls
- **Applied Cryptographer** — designing and implementing secure communication systems

## Who This Pathway Suits

This pathway is best suited to students who:

- Are genuinely interested in how systems can be broken — not just built
- Have a natural instinct for thinking adversarially — "how could this be exploited?"
- Are detail-oriented and methodical — security work requires precision and patience
- Are comfortable with networking concepts and want to go deep into protocols and systems
- Are interested in the legal and ethical dimensions of technology — this field operates within strict legal boundaries
- Enjoy investigative work — digital forensics in particular suits analytical, evidence-driven thinkers
- Are drawn to a field with clear professional certifications (CEH, OSCP, CISSP) and strong industry demand

Strong security awareness and analytical thinking are the primary indicators of fit for this pathway. Programming ability matters but is less central than in AI — the focus is on understanding and manipulating systems, not building new ones from scratch.

---

## How This Pathway Differs from the Others

Compared to AI, this pathway focuses on attacking, defending, and investigating existing systems rather than building intelligent ones. The mathematical requirements are lower, but the depth of systems knowledge required — particularly networking and operating systems — is significantly higher.

Compared to Computing/Software Engineering, this pathway sacrifices breadth in application development (web, mobile, UX) for depth in security-specific skills. A Cybersecurity graduate knows far more about how systems fail than a Computing graduate — but knows less about building production software.

This is a specialist pathway for students who know they want to work in security. The career outcomes are specific, well-compensated, and in extremely high demand globally.
